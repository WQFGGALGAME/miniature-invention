# 滚动歌词网页工具 (Karaoke Lyrics Web Tool) Spec

## Why
用户需要一个可以通过加载本地音频和歌词文件，来播放音乐并展示滚动歌词的网页工具。该工具必须具备类似卡拉OK的逐字高亮功能，允许自定义背景色和字体颜色，且最重要的需求是UI必须完美还原甚至超越用户提供的深色音乐播放器参考图。

## What Changes
- 创建一个全新的网页应用（HTML/CSS/JS 或轻量级前端框架）。
- 实现精美的、响应式的UI界面，高度还原提供的参考图（包括顶部导航、歌曲信息、居中的歌词滚动区、底部播放控制栏和进度条）。
- 实现音频播放器核心逻辑（播放/暂停、进度更新、时长显示）。
- 实现歌词解析器：支持读取标准LRC文件，并实现或模拟逐字高亮的卡拉OK效果。
- 实现自定义设置面板：允许用户修改背景颜色、非激活歌词颜色、高亮歌词颜色和字体大小。
- 支持拖拽上传音频和歌词文件。
- 支持键盘快捷键操作（如空格键播放/暂停，左右方向键快进/快退）。
- 响应式布局适配不同屏幕尺寸。
- 提供歌词时间轴微调校准与编辑功能，实现精确同步。

## Impact
- Affected specs: 增强现有播放器功能，提升用户体验。
- Affected code: `index.html`, `style.css`, `app.js` 将新增相应逻辑。

## ADDED Requirements
### Requirement: 完美还原参考图UI
系统必须提供一个与参考图一致的深色主题UI界面。

#### Scenario: 界面加载与展示
- **WHEN** 用户打开网页时
- **THEN** 呈现参考图中的排版布局（包括上下工具栏、图标占位符、歌词区域），并提供优雅的文件上传入口（可设计为初始屏幕或隐藏在菜单中）。

### Requirement: 卡拉OK逐字高亮歌词
系统必须支持歌词随音频同步滚动，并且当前行的歌词要逐字高亮显示。

#### Scenario: 播放音乐时
- **WHEN** 音乐正在播放
- **THEN** 歌词容器平滑滚动使当前行居中，并且当前行的文字根据音频进度呈现逐字填充的高亮效果。

### Requirement: 自定义颜色主题
系统必须允许用户自定义背景色和字体颜色。

#### Scenario: 用户调整颜色
- **WHEN** 用户在设置面板中选择新的背景色或字体颜色
- **THEN** 整个播放器界面的背景和歌词颜色实时更新。

### Requirement: 增强交互与适配
系统必须支持快捷键、拖拽上传，并在多设备上表现良好。

#### Scenario: 拖拽文件上传
- **WHEN** 用户将音频或歌词文件拖拽到网页区域
- **THEN** 系统解析并加载对应文件。

#### Scenario: 快捷键操作
- **WHEN** 用户按下空格键或左右方向键
- **THEN** 触发播放/暂停，或快进/快退。

### Requirement: 歌词时间轴校准
系统必须提供修改时间轴的工具。

#### Scenario: 用户微调歌词时间
- **WHEN** 用户在时间轴编辑模式下调整某行歌词的延迟/提前
- **THEN** 歌词同步时间实时更新并保存。
