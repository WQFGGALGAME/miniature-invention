# Tasks
- [x] Task 1: 项目初始化与基础结构搭建。创建 `index.html`、`style.css`、`app.js` 等基础文件，引入必要的图标库（如 FontAwesome 或内置 SVG）。
- [x] Task 2: 核心UI布局实现。根据参考图，使用 CSS Grid/Flexbox 完美还原深色主题界面，包括顶部栏、歌词展示区、底部播放控制台（进度条、播放按钮组、底部导航栏）。
- [x] Task 3: 文件加载逻辑。实现本地音频文件和歌词文件（LRC）的读取功能，在界面中提供一个不破坏整体美感的上传入口。
- [x] Task 4: 歌词解析与卡拉OK高亮引擎。解析LRC文件，结合音频的 `timeupdate` 事件，实现歌词的平滑滚动。针对逐字高亮，如果是标准LRC，根据每行的持续时间和字数计算单字高亮时间，实现平滑的文字填充效果。
- [x] Task 5: 颜色自定义功能。添加一个隐藏/弹出的设置面板，提供颜色选择器修改背景颜色和歌词字体颜色，并通过 CSS 变量实时生效。
- [x] Task 6: 细节打磨与动效优化。优化滚动动画的流畅度，调整文字大小和字重以完美匹配参考图，确保按钮有交互反馈。
- [x] Task 7: 字体大小调节。在设置面板中增加字体大小调节滑块，动态改变全局或歌词的字体大小。
- [x] Task 8: 拖拽上传支持。监听网页拖拽事件，允许用户直接拖入音频文件和 LRC 歌词文件进行加载。
- [x] Task 9: 键盘快捷键操作。绑定键盘事件，实现空格键播放/暂停，左右方向键快进/快退。
- [x] Task 10: 响应式布局优化。使用媒体查询调整在手机、平板和桌面端的显示比例，确保在不同屏幕下良好适配。
- [x] Task 11: 时间轴微调校准与编辑功能。在界面提供"校准模式"，允许用户通过按钮整体提前/延后歌词，甚至修改单行歌词的时间点。
- [x] Task 12 (极致优化): 数据持久化。使用 LocalStorage 保存用户的颜色、字号、音量和时间轴微调偏好，刷新不丢失。
- [x] Task 13 (极致优化): 性能与内存优化。在切换音频文件时使用 `URL.revokeObjectURL` 释放旧的 Blob 内存，并增加 `<audio>` 错误边界处理。
- [x] Task 14 (极致优化): Web Audio API 动感频谱可视化。在背景层添加 Canvas，提取音频频率数据绘制随音乐跳动的底部频谱。
- [x] Task 15 (极致优化): 沉浸式全屏模式。通过 Fullscreen API 在左上角添加全屏切换功能，打造完美的纯净播放体验。
